This file maps each required Vercel or bridge variable to its source. Do not paste real passwords, API keys, merchant keys, or service-role keys into this repository.

## INET Cloud Database Gap

The received INET details describe a VM, not a database service. For database cutover, request one of these from INET:

- Managed PostgreSQL endpoint: host, port, database name, username, password, SSL mode.
- Or permission to install PostgreSQL on VM `SST-Innovation-01-POC`.
- Confirmation whether Supabase Auth, PostgREST/Data API, Realtime, Storage, and RLS equivalents are required.

Important: the current app imports Supabase clients directly. A plain PostgreSQL `DATABASE_URL` alone will not replace `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, and `SUPABASE_SERVICE_ROLE_KEY` without additional backend refactoring.
